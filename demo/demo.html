Easytype Plugin for JQuery

Switch Language: English | Chinese

Easytype plugin restricts user's input to a pre-defined pattern. It guides users to type legal characters only and ensures that the input follows the format.

Usage

Easytype defines a simple convention for a mask pattern, which consists of masks and fixed text fragments. For masks, # is for numbers, $ could be for text (technically an alpha-numeric value), any other character is taken literally as a fixed text fragment.

Examples:

  • A Social Security Number mask: ###-##-####.
  • A US phone mask: could be (###) ###-####.
  • A date: ##/##/####.
  • Software Serial Number: $$$$-$$$$-$$$$-$$$$.
  • Any pattern you need!: If(####){$$$$$$}eLsE{$$$$$$}.

etc..

Before Easytype plugin is enabled, the code for the plugin must be imported first. Usually, it appears after the importation of JQuery library.

To apply a pattern to an input control, after acquiring that input control using JQuery selector, simply call easytype() method and toss over the pattern you need as a string.

Example:

$('#phone_number').easytype('(###) ###-####');

Try it now!

(Note: you can only type numeric values in this example!)

To get the value of the input control, just use $('#phone_number').val(). Nothing changed!

Features

Colorful rendering! To make it easier to identify mask and fixed text entries, Easytype can colorize the characters differently, even within the same input control!

To enable this feature, pass an object-type parameter when calling easytype() method, and specify deluxe attribute as true in conjunction with pattern attribute set to the preferred pattern. Then you have to define css style for

  • easytype_prompt: Style for the ghostly shadow prompt underneath the input control.
  • easytype_input_text: Style for input text in the input control.
  • easytype_label: Style for the fixed text fragment entries.
  • easytype_mask: Style for the masked entries.

Example:

.easytype_prompt {
    color: #dcdcdc;
}

.easytype_input_text {
    color: green;
}

.easytype_input_text .easytype_label {
    color: #dc143c;
}

$('#ISBN').easytype({deluxe: true, pattern:'ISBN ###-#-##-######-#'});

Try it now:

Don't like the css class names? No problem, define them yourself by adding more options to the object parameter of easytype() method

Example:

$('#ISBN').easytype({deluxe: true, pattern:'ISBN ###-#-##-######-#',
                    css_prompt: 'my_prompt_class',
                    css_entry_label: 'my_label_class',
                    css_entry_mask: 'my_mask_class',
                    css_input_text: 'my_input_text_class'});

Unobtrusive to the original layout  Worry about Easytype plugin has modified your inputbox and might jeopardize your page layout? Please don't worry. Easytype tries very hard to maintain the same layout context around your inputbox. The margin and border and everything will just be kept same. Simply call easytype() method with no extra care taken! No hassle typing! As you might have noticed, the fixed text fragments are automatically filled into the input control for convenience. However some users might be unaware of this fact and just input as they do with a normal input control. This is definitely not a problem! As long as a user tries to type characters that match what ghostly shadow prompt shows, they will be recognized correctly and the caret will move normally just as it does with a normal input control. However, if the user decides to input a masked entry after the fixed text fragment, the caret will automatically jump to the position of that entry.
